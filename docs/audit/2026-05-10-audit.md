## Threat model

Unchanged from the 2026-04-26 audit. Agnostik is a types library
consumed by 11 AGNOS components (daimon, hoosh, agnoshi, aegis,
argonaut, sigil, ark, kavach, stiva, nein, yukti). The
attack surface is **untrusted JSON / version / UUID / traceparent
input** crossing the serde boundary into agnostik parsers. Output
flows back to consumers as `Result<value, AgnostikError>` or as
typed structs. agnostik has no syscall surface that interacts with
external state beyond `getrandom`/`/dev/urandom` (CSPRNG) and
`stderr` (fatal-error message).

## External research (CVE / 0-day scan)

- **JSON `\u` decoders** are a frequent CVE class (e.g.
  CVE-2017-12635 in CouchDB's JSON parser — duplicate-key hash
  collision; CVE-2022-23529 in jsonwebtoken — missing key type
  validation; CVE-2024-27982 in Node — HTTP request smuggling via
  unicode normalization). agnostik's decoder explicitly **does not
  decode** to a normalized form — it emits raw UTF-8 bytes from the
  unicode codepoint. Surrogate handling matches RFC 7159 § 7;
  malformed sequences emit U+FFFD per common JSON-parser convention.
- **PII categorization** is policy-flavored, not a CVE class. The
  3 new variants (`PII_GENETIC`, `PII_BIOMETRIC_TEMPLATE`,
  `PII_PRECISE_GEOLOCATION`) are additive enum tags — no parser
  surface, no codegen change. No security implication.
- **CSPRNG patterns** unchanged from the 2026-04-26 audit (F-001
  closure stands).

## Findings

### F-012 — `_fill_random` fatal-message stderr write off-by-one (INFO)

**Severity:** **INFO** — cosmetic. The fatal-error message at
`src/types.cyr:27` is 68 bytes; the syscall passes 67. The trailing
newline is silently dropped from stderr output. No memory-safety
issue, no information leak — the missing byte is just `\n`.

**Repro:**
```cyrius
syscall(1, 2, "agnostik: CSPRNG unavailable (getrandom + /dev/urandom both failed)\n", 67);
                                                                                       ^^
                                                                            literal is 68 bytes
```

`echo -n "agnostik: CSPRNG unavailable (getrandom + /dev/urandom both failed)\n" | wc -c` → 68.

**Reachability:** the message fires only when **both** `getrandom(2)`
syscall returns short AND `/dev/urandom` open/read fails. This path
is exercised only on extreme sandbox failure (seccomp blocking both
syscalls + the fallback file path). Real-world reachability is
near-zero; the fatal-stderr-then-exit shape itself is correct.

**Fix:** swap the hardcoded byte count for `strlen()` so the count
tracks the literal automatically. Eliminates the off-by-one and
prevents future drift if the message text is ever edited.

```cyrius
var msg = "agnostik: CSPRNG unavailable (getrandom + /dev/urandom both failed)\n";
syscall(1, 2, msg, strlen(msg));
```

**Disposition:** fixed in this slot (v1.0.8). No CVE, no advisory.

## What we verified clean (this pass)

### `\uXXXX` decoder (`src/types.cyr:_json_str`, v1.0.7) — new surface

- **Buffer sizing:** `out = alloc(slen + 1)` where `slen` is the
  full source `src.len`, not the value length. Each `\uXXXX` is 6
  source bytes producing 1-3 UTF-8 bytes (BMP) or 3 bytes (U+FFFD
  fallback); each surrogate pair is 12 source bytes producing 4
  UTF-8 bytes. Output ≤ input in every path; the buffer is
  conservatively over-allocated. Tightest case (truncated `\u` at
  EOF emitting U+FFFD = 3 bytes) verified to stay within `slen + 1`.
- **Bounds checks:** every `load8(data + offset)` access is gated
  by `offset < slen`. Specifically, `vi + 6 > slen` guards the
  single-codepoint path (4 hex digits at `vi+2..vi+5`); `vi + 12 > slen`
  guards the surrogate-pair path (4 hex digits at `vi+8..vi+11`).
- **Surrogate validation:** high-surrogate range `[0xD800, 0xDBFF]`
  routes to pair-handler; low-surrogate `[0xDC00, 0xDFFF]` orphan
  path emits U+FFFD; only the post-check in pair-handler accepts a
  valid low (`lo >= 0xDC00 && lo <= 0xDFFF`). Invalid hex in `lo`
  returns -1 from `_json_parse_u4` which falls through the
  `lo < 0xDC00` branch to U+FFFD. **No surrogate codepoint is ever
  emitted as raw UTF-8** (which would be invalid per RFC 3629).
- **Integer overflow:** `0x10000 + ((cp - 0xD800) << 10) + (lo - 0xDC00)`
  — cp range yields shift values 0..1023, multiplied by 1024 = 0..1047552.
  Plus lo-DC00 0..1023. Total max = 1048575 + 0x10000 = 0x10FFFF.
  Within i64 range; no overflow.

### Syscall review (`src/types.cyr`, `src/error.cyr`)

- `syscall(318, buf, n, 0)` — `getrandom(2)`. `n` is bounded (16
  for UUID, 8 for span). `buf` is allocator-owned. ✓
- `syscall(2, "/dev/urandom", 0, 0)` — `open(2)` with literal path,
  no user-controllable component. ✓
- `syscall(0, fd, buf + off, n - off)` — `read(2)`. fd from
  `open(2)`, buf is allocator-owned, off bounded `0..n`. ✓
- `syscall(3, fd)` — `close(2)`. Always safe. ✓
- `syscall(1, 2, msg, len)` — `write(2)` to stderr. F-012 finding
  on the byte count above. Fixed in v1.0.8.
- `syscall(60, 70)` — `exit(2)` with literal exit code. ✓
- **No `sys_system`, `exec*`, `popen` anywhere in src/.** ✓

### Buffer safety

- **0 stack-allocated `var X[N]` buffers in src/.** All buffers heap-
  allocated via `alloc(N)` with literal `N` matching the struct
  layout (e.g. `alloc(48)` for ResourceLimits 6×8B). ✓

### Path traversal

- agnostik **models** filesystem paths (`FilesystemRule.path`,
  `LandlockFsRule.path`, `NamespaceEntry.path`) but does not
  open/read/traverse them. Path-traversal mitigation is the
  consumer's responsibility (kavach for sandbox enforcement;
  daimon for runtime checks). agnostik's only fs syscall is
  `open("/dev/urandom")` with a literal path. ✓

### Pointer validation

- 42 `load8(data + offset)` / `store8(out + op)` sites in
  `src/types.cyr` (the parser hot-spot module). Every call's
  offset bounded by the loop's `vi < slen` / `op < buffer_size`
  invariant. ✓
